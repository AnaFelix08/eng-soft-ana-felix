**Histórias de Usuário**
--- 

- Como dono do hotel, quero visualizar todos os quartos em uma tela para saber quais estão ocupados, livres ou sujos.
- Como recepcionista, quero cadastrar hóspedes rapidamente para agilizar o check-in.
- Como dono do hotel, quero evitar que dois hóspedes ocupem o mesmo quarto para evitar erros e constrangimentos.
- Como dono do hotel, quero registrar reservas para organizar a ocupação.
- Como dono do hotel, quero buscar hóspedes por nome ou CPF para agilizar o atendimento de clientes recorrentes.
- Como funcionário (Tião), quero registrar consumo rapidamente pelo celular para não esquecer pedidos.
- Como dono do hotel, quero registrar consumo vinculado ao quarto para cobrar corretamente.
- Como sistema, quero registrar data, hora e responsável por cada consumo para garantir confiabilidade.
- Como camareira (Dona Maria), quero visualizar quais quartos precisam de limpeza para trabalhar sem depender do gerente.
- Como dono do hotel, quero controlar o status dos quartos (ocupado, livre, sujo) para organizar operações.
- Como dono do hotel, quero calcular automaticamente o valor total da estadia para evitar erros.
- Como dono do hotel, quero visualizar o faturamento do período para saber se tive lucro.
- Como dono do hotel, quero alterar os preços dos quartos facilmente para adaptar a demanda.

**Requisitos Funcionais**
---

**RF01:** O sistema deve exibir painel com status dos quartos (livre, ocupado, sujo)
**RF02:** O sistema deve permitir cadastro e busca de hóspedes por nome ou CPF
**RF03:** O sistema deve registrar reservas
**RF04:** O sistema deve impedir duplicidade de ocupação de quartos
**RF05:** O sistema deve permitir cadastro de tipos de quartos com preços diferentes
**RF06:** O sistema deve permitir alteração de preços pelo usuário
**RF07:** O sistema deve registrar consumo por quarto
**RF08:** O sistema deve registrar data, hora e responsável por cada consumo
**RF09:** O sistema deve calcular automaticamente o valor total da estadia
**RF10:** O sistema deve permitir controle de limpeza dos quartos
**RF11:** O sistema deve gerar resumo financeiro (faturamento)

**Requisitos Não Funcionais**
---

**RNF01:** O sistema deve ser simples e intuitivo
**RNF02:** Deve funcionar em computadores comuns
**RNF03:** Deve permitir uso em celular para lançamentos simples
**RNF04:** Deve ter resposta rápida (tempo real)
**RNF05:** Deve ser confiável (dados não podem ser perdidos)
**RNF06:** Deve ser utilizável por pessoas com baixo nível de conhecimento tecnológico
**RNF07:** Deve funcionar mesmo com instabilidade de internet (quando possível)
